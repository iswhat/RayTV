# RayTV 混合源架构设计文档

## 一、核心设计理念

### 1.1 混合源使用模式
用户可以自由组合不同网络源的内容:
- 点播内容: 使用源A的站点列表
- 直播内容: 使用源B的直播频道
- 壁纸资源: 使用源C的壁纸
- 广告过滤: 合并使用所有源的过滤规则
- 解码配置: 使用所有源的解码信息

### 1.2 存储模型
每个网络源被独立存储后,其内容被提取到统一的使用池中:

```
网络源 (NetworkSource)
├── 点播站点列表 → 提取到 VodSitePool
├── 直播频道列表 → 提取到 LiveChannelPool
├── 壁纸资源 → 提取到 WallpaperPool
├── 广告过滤规则 → 合并到 GlobalAdBlockRules
├── 解码配置 → 合并到 GlobalDecoderConfigs
└── 解析器配置 → 提取到 ParserConfigPool
```

## 二、数据结构设计

### 2.1 网络源配置 (NetworkSourceConfig)
```typescript
interface NetworkSourceConfig {
  id: string;                    // 唯一标识
  name: string;                  // 显示名称
  url: string;                   // 配置源URL
  type: 'json' | 'yaml' | 'txt';
  version: string;               // 版本号
  isActive: boolean;             // 是否启用
  
  // 内容开关 - 控制哪些内容被提取到使用池
  contentUsage: {
    vodSites: boolean;           // 是否使用点播站点
    liveChannels: boolean;       // 是否使用直播频道
    wallpapers: boolean;         // 是否使用壁纸
    adBlockRules: boolean;       // 是否使用广告过滤规则
    decoderConfigs: boolean;     // 是否使用解码配置
    parserConfigs: boolean;      // 是否使用解析器配置
  };
  
  // 优先级配置 (1-10, 数字越大优先级越高)
  priority: number;
  
  // 使用池中的标签/标记 (用于UI显示来源)
  tags?: string[];
  
  lastUpdated: number;           // 最后更新时间
  healthStatus: 'healthy' | 'degraded' | 'unreachable';
}
```

## 三、服务架构重构

### 3.1 新增服务: SourcePoolManager

```typescript
class SourcePoolManager {
  private static instance: SourcePoolManager;
  
  // 统一使用池
  private vodPool: VodSitePool;
  private livePool: LiveChannelPool;
  private wallpaperPool: WallpaperPool;
  private adBlockRules: GlobalAdBlockRules;
  private decoderConfigs: GlobalDecoderConfigs;
  
  // 构建函数
  public async loadFromSource(source: NetworkSourceConfig): Promise<void> {
    if (!source.isActive) return;
    
    const usage = source.contentUsage;
    
    // 根据内容开关选择性加载
    if (usage.vodSites) {
      const vodSites = await this.loadVodSites(source);
      this.vodPool.addSites(vodSites);
    }
    
    if (usage.liveChannels) {
      const liveChannels = await this.loadLiveChannels(source);
      this.livePool.addChannels(liveChannels);
    }
    
    if (usage.wallpapers) {
      const wallpapers = await this.loadWallpapers(source);
      this.wallpaperPool.addWallpapers(wallpapers);
    }
    
    if (usage.adBlockRules) {
      const rules = await this.loadAdBlockRules(source);
      this.adBlockRules.addRules(rules, source.id);
    }
    
    if (usage.decoderConfigs) {
      const configs = await this.loadDecoderConfigs(source);
      this.decoderConfigs.addConfigs(configs, source.id);
    }
  }
  
  // 重新构建所有池
  public async rebuildAllPools(sources: NetworkSourceConfig[]): Promise<void> {
    // 清空所有池
    this.vodPool.clear();
    this.livePool.clear();
    this.wallpaperPool.clear();
    this.adBlockRules.clear();
    this.decoderConfigs.clear();
    
    // 从所有启用的源重新加载
    const activeSources = sources.filter(s => s.isActive);
    for (const source of activeSources) {
      await this.loadFromSource(source);
    }
  }
}
```

### 3.2 ConfigSourceService 职责变更

**旧职责**:
- 直接提供getSites(), getLive(), getWallpapers()等方法
- 每次调用时从单个源获取数据

**新职责**:
- 管理网络源配置(添加、删除、更新、启用/禁用)
- 提供源的健康检查
- 触发SourcePoolManager重建池
- 不再直接提供内容访问方法

**新的API设计**:
```typescript
// 网络源管理
addSource(source: NetworkSourceConfig): Promise<void>
removeSource(sourceId: string): Promise<void>
updateSource(source: NetworkSourceConfig): Promise<void>
toggleSource(sourceId: string, isActive: boolean): Promise<void>
updateContentUsage(sourceId: string, usage: ContentUsage): Promise<void>

// 源信息查询
getAllSources(): NetworkSourceConfig[]
getActiveSources(): NetworkSourceConfig[]
getSourceById(sourceId: string): NetworkSourceConfig | null

// 健康检查
checkSourceHealth(sourceId: string): Promise<HealthStatus>
checkAllSourcesHealth(): Promise<Map<string, HealthStatus>>

// 池重建触发
rebuildPools(): Promise<void>
onSourceChanged(): void  // 自动触发重建
```

## 七、优势总结

1. **灵活性**: 用户可以自由组合不同源的内容
2. **可维护性**: 源的添加/修改不影响其他源
3. **透明性**: 清晰显示每个内容的来源
4. **性能优化**: 池化访问,避免重复加载
5. **容错性**: 单个源故障不影响其他源的使用
6. **用户控制**: 用户可以精确控制每个源的内容使用
