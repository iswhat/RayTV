# RayTV 壳类型视频聚合器项目说明

## 项目来源

- **AndroidVer**：来自于 [https://github.com/FongMi/TV](https://github.com/FongMi/TV)
- **RayTV**：参考FongMi/TV，基于Harmony OS进行的全新开发实现

## 项目概述

RayTV是一个基于Harmony OS开发的专业壳类型视频聚合器，采用配置驱动的内容聚合架构。该应用不直接存储视频内容，而是通过动态配置源获取和整合来自多个第三方平台的视频资源，为用户提供统一的观看体验。

**重要声明：**
- 本项目仅用于Harmony OS程序开发学习目的
- 同时作为对Trae cn AI IDE能力的测试与验证
- 程序本身不具备也不集成任何站点的解析处理能力
- 项目通过配置源管理实现内容聚合，不直接处理视频内容
- 项目开发严格遵守相关法律法规，避免出现内容的版权问题

## 主要功能

作为学习用的壳类型视频聚合器，主要功能模块包括：
- 🎯 **配置驱动架构**：动态加载和管理多个视频内容源
- 🔄 **内容智能聚合**：统一解析和整合不同平台的视频资源
- 🧠 **AI智能推荐**：基于用户行为的个性化内容推荐
- ▶️ **增强播放体验**：自适应流媒体、画质优化、智能缓存
- 🔍 **跨源统一搜索**：一次搜索遍历所有配置源
- 🎛️ **插件化解析**：支持多种内容格式的可扩展解析器
- 🛡️ **智能广告拦截**：自动识别和过滤广告内容
- 👤 **用户画像系统**：学习用户偏好，提供个性化体验
- 📊 **实时性能监控**：系统性能指标收集和分析
- 🎨 **响应式UI设计**：可复用组件和现代化界面
- 🤖 **Harmony OS集成**：系统特有功能完整演示

## 技术栈

- **Harmony OS**：应用开发框架
- **ArkTS**：主要开发语言（HarmonyOS特有）
- **TypeScript**：辅助开发语言
- **Ets**：Harmony OS UI组件
- **核心架构**：依赖注入 + 事件总线 + 服务层
- **现代化架构模式**：MVVM + 插件化架构
- **配置管理**：动态配置源加载机制
- **内容聚合**：多源内容整合引擎

## 项目结构

### AndroidVer目录

包含基于Android平台的实现，来源于FongMi/TV项目，仅作为需求参考。

### RayTV目录

包含基于Harmony OS的实现，是本项目的核心部分，采用全新的架构设计：

```
raytv/
├── common/                    # 通用组件和工具
│   ├── di/                   # 依赖注入容器
│   ├── event/                # 事件总线系统
│   └── util/                 # 工具类（包括性能监控）
├── service/                  # 服务层
│   ├── config/               # 配置源管理服务
│   │   └── ConfigSourceService.ts
│   ├── content/              # 内容聚合服务
│   │   └── ContentAggregator.ts
│   ├── parser/               # 解析器管理服务
│   │   └── ParserManager.ts
│   ├── recommendation/       # 智能推荐服务
│   │   └── RecommendationEngine.ts
│   ├── playback/             # 增强播放服务
│   │   └── EnhancedPlaybackService.ts
│   ├── ads/                  # 广告拦截服务
│   │   └── EnhancedAdBlockService.ts
│   └── interfaces/           # 服务接口定义
├── types/                    # 统一数据模型
│   └── MediaContent.ts
├── test/                     # 集成测试
│   └── InfrastructureIntegrationTest.ts
└── App.ets                   # 应用入口
```
